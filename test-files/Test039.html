Select a category from the left to manage it. You can create a new one with the New Category button.
Important: This manager will add/update documents in your existing Firestore structure. It is designed to be compatible with your current public page. See instructions below.
Security note: this manager performs writes and will call your Supabase Edge Function to upload/delete files. Protect access using Firebase Auth + Rules or host it behind an admin-only route.

How it works (short): • Categories = documents in collection: portfolio.

Each document may have fields: title, description. • Images = documents in portfolio/{catId}/images with fields:

url (string) - public HTTP URL returned from Supabase

filename (string) - storage path/filename used on Supabase

rank (number) - controls ordering; lower = earlier

caption (string, optional)

createdAt (timestamp)


Supabase edge function contract (required): • Upload endpoint (POST) receives JSON { path: "portfolio/{catId}/images/{filename}", base64 }

returns { publicUrl: "https://..." } • (Optional) Delete endpoint (POST) receives JSON { path: "portfolio/{catId}/images/{filename}" }

returns { ok: true }


Public page change (to pick cover deterministically): db.collection('portfolio').doc(docId).collection('images').orderBy('rank').limit(1).get()

Keep your Edge Function secure — it should verify requests (App Check token or a secret) before uploading to Supabase.
